Rich Internet Application Coding Rules for Adobe Flex


This document tries to describe some basic rules in how to create consistent Rich Internet Applications written in Adobe Flex 3. In addition, the coding rules of Adobe Flex 4 will also be considered.


1. General rules:


- Use coding conventions:

http://opensource.adobe.com/wiki/display/flexsdk/Coding+Conventions

- Write documentation for classes, functions and properties

- Think about your team ... you are not alone!

- Try to work in pairs (2 developers) ... see Extreme Programming:

http://en.wikipedia.org/wiki/Extreme_Programming#Practices


2. Flex developer rules:


- When creating Flex applications, always think in States (View States, Component States, User States)

- Always think about reusable code ... modular applications


2.1. Swiz MVC rules:


- Whenever you will find a new datatype in your application, create a new Value Object (VO) 

- Do not create redundancy inside the Models (e.g. do not store the same data twice) and always try to reduce the amount of data stored in the Models

- Write Event / Command - pairs with the same name ... e.g. CreateUserEvent / CreateUserCommand


2.2. MXML document rules:


- Try to prevent the <mx:Script/>-tag in MXML-files ... the logic should be placed in the Controllers

- Prevent [Bindable] properties in MXML-files ... put them into your Models

- Use a consistent MXML layout:

1. <mx:State/>

2. <mx:Transition/>

3. <mx:Script/>

3.1. Constants

3.2. Properties

3.3. Constructors

3.4. Methods / Functions

4. <mx:Effect/>

5. <mx:View/>

- Use a consistent attribute layout inside of MXML nodes:


<mx:Canvas

xmlns:mx="http://www.adobe.com/2006/mxml"   // namespaces

id="bx_myBox"   // component id

x="0" y="0"   // layout information

width="100%" height="100%"

backgroundAlpha="0.5"   // styling

backgroundColor="#ff00ff"

styleName="myBoxStyle"

showEffect="{ fx_fadeIn }"   // effects

click="onClick(event)"   // events

>

<mx:Repeater

id="rp_myRepeater"   // component id

dataProvider="{ model.repeaterData }"    // component values

>

<mx:Image

source="{ PersonVO( rp_myRepeater.currentItem ).profileImage }"

/>

</mx:Repeater>

</mx:Canvas>


<mx:List 

id="li_myList"   // component id

width="100%" height="100%"   // layout information

dataProvider="{ model.listData }"    // component values

columnCount="1"

rowCount="2"

useRollOver="false"

variableRowHeight="true"

horizontalScrollPolicy="off"

verticalScrollPolicy="off"

borderStyle="none"   // styling

selectionColor="#ffffff"

styleName="myListStyle"

hideEffect="{ fx_fadeOut }"   // effects

dragStart="onDragStart(event)"   // events

/>


- Always try to prevent styling attributes like backgroundColor="#ff00ff" in MXML nodes. Put these properties into CSS files and link them via styleName="myListStyle" into the MXML node.

- Use a consistent naming convention for the id attributes in MXML nodes:


<mx:Text

id="txt_myText"

/>

<mx:TextInput

id="tip_myTextInput"

/>

<mx:TextArea

id="ta_myTextArea"

/>

<mx:CheckBox

id="cb_myCheckBox"

/>

<mx:ComboBox

id="cbb_myComboBox"

/>

<mx:Button

id="btn_myButton"

/>

- Use a consistent AS layout:

1. Constants

2. Properties

3. Constructors

4. Methods / Functions
