Login Example – Specification Document

This document describes the basic architecture of the Login Example application. You will get an overview about the application flow and states. Furthermore, types of value objects (VOs) and client-server interface (delegates) will be discussed.




1.1. Application Flow and States

1.2. Value Objects

The Login Example application uses the following value objects.




SettingsVO
 - layouts : ArrayCollection <LayoutVO>

LayoutVO
 - name : String
 - width : Number
 - heigth : Number
 - stateName : String
 - styles : ArrayCollection <StyleVO>
 - locales : ArrayCollection <LocaleVO>

LocaleVO
 - name : String
 - code : String

StyleVO
 - name : String
 - path : String

AuthenticationVO
 - username : String
 - password : String
 - email : String

UserVO
 - username : String
 - firstname : String
 - lastname : String
 - email : String
 - birthday : Date
